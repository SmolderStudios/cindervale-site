Read me first

This is the running list of 3D models for SLOTROOM — a PS1/PSX-style horror slot-machine game (walk around a grimy room, play a cursed slot machine, survive the nights). Each card below is one model, with a ready-to-paste prompt, a size/poly target, and the exact filename to hand back.

Click the status button on a card to cycle it Not started → Working on it → Delivered. Status is shared — everyone looking at this page sees the same thing, and it updates within a few seconds (the badge up top shows when you're connected). Finished .glb files still go in the shared Drive folder.

Start with the P0 cards — the characters and the slot machine are where real models help most. P1 charms are small collectible props. P2 are environment dressing — nice-to-have, the game already has placeholder versions.

How this works — Troper only does 4 things

These are made in Tripo, so no Blender skill needed. For each card:

  • 1 · Generate it in Tripo — paste the card's prompt into text-to-3D, or (usually better) drop the card's reference image into Tripo's image-to-3D.
  • 2 · Export as .glb — that's the format the game loads. (GLB, not FBX or OBJ.)
  • 3 · Name the file exactly what the card says, lowercase — e.g. prisoner.glb.
  • 4 · Drop it in the Drive folder → shared Google Drive folder ↗.

Don't worry about any of this — the game fixes it automatically on import

Troper does not need to touch these in Blender. Claude handles them in code when the model goes in:

  • Size, position & facing — auto-scaled to the right size, centered, and turned to face the room. Whatever scale/rotation Tripo spits out is fine.
  • The retro PS1 look — flat shading, crunchy low-res textures and the wobble are all added on the way in. Tripo's smooth, shiny, modern output is expected; it gets crushed into the style automatically.
  • Shiny / PBR / normal maps — ignored by the game, so it doesn't matter if Tripo bakes them in.
  • Animation IS wanted for the characters now — anything with a 🎬 needs animation tag should come rigged with animation clips (Tripo's rig + animate). A bare rig with no clip is still frozen, so include at least an idle (and a walk on the prisoner). The game auto-plays them. Everything without the tag stays static — Claude animates those in code (doors, lever, etc.).

The only two things that actually matter for the model

  • Make it the right thing — match the card's description. If Tripo offers a lower-poly / cleaner option, take it; if not, whatever it gives is fine.
  • The slot cabinet is the one exception — leave its reel window and the lit belly panel as flat blank recesses; the game draws the moving screens onto them.

Reference images will fill in on the cards as they're added, so image-to-3D is the smoothest path.
